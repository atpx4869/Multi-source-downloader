# 常见问题

## 安装问题

### Q: 如何安装依赖？
**A**: 双击运行 `scripts\首次安装.bat`，或手动运行：
```bash
pip install -r requirements.txt
```

### Q: Windows 7 安装失败？
**A**: 使用专用依赖文件：
```bash
pip install -r requirements_win7.txt
```

### Q: 提示缺少模块？
**A**: 
1. 确认虚拟环境已激活
2. 重新安装依赖
3. 检查Python版本（需要3.8+）

---

## 运行问题

### Q: 程序无法启动？
**A**: 
1. 检查Python是否正确安装
2. 查看错误日志
3. 尝试清理缓存后启动
4. 确认所有依赖已安装

### Q: 启动很慢？
**A**:
1. 首次启动需要2-3秒初始化
2. 清理缓存：`scripts\启动应用（清理缓存）.bat`
3. 检查杀毒软件是否拦截

### Q: 界面显示异常？
**A**:
1. 检查显示缩放设置
2. 更新显卡驱动
3. 尝试兼容模式运行

---

## 搜索问题

### Q: 搜索无结果？
**A**:
1. **检查标准号格式**：
   - 正确: `GB/T 3324`
   - 错误: `GBT3324` 或 `GB T 3324`
2. **尝试其他数据源**：不同源数据可能不同
3. **检查网络连接**
4. **增加超时时间**：编辑 `config/api_config.json`

### Q: 搜索很慢？
**A**:
1. ZBY源使用浏览器模拟，较慢（约10秒）
2. 优先使用GBW或BY源
3. 检查网络速度
4. 减少搜索结果数量

### Q: 部分源搜索失败？
**A**:
1. 某些源可能临时不可用
2. 检查配置文件中该源是否启用
3. 查看日志了解详细错误
4. 使用其他可用源

---

## 下载问题

### Q: 下载失败？
**A**:
1. **检查网络连接**
2. **尝试其他数据源**：
   - 优先级：BY > GBW > ZBY
3. **检查磁盘空间**
4. **查看错误日志**

### Q: 下载速度慢？
**A**:
1. 取决于源服务器速度
2. 检查本地网络带宽
3. 尝试其他数据源
4. 避免高峰时段

### Q: 下载的文件打不开？
**A**:
1. 检查文件是否完整下载
2. 确认文件格式（PDF需要PDF阅读器）
3. 重新下载
4. 尝试其他数据源

### Q: 批量下载中断？
**A**:
1. 程序支持断点续传
2. 重新启动下载任务
3. 检查网络稳定性
4. 减少并发下载数量

---

## Excel功能问题

### Q: Excel导入失败？
**A**:
1. 确认文件格式（支持.xlsx, .xls, .csv）
2. 检查文件是否被其他程序占用
3. 确认标准号列存在
4. 检查文件编码（CSV需要UTF-8）

### Q: Excel导出失败？
**A**:
1. **关闭所有打开的Excel文件**
2. 检查目标目录写入权限
3. 确认磁盘空间充足
4. 尝试更换导出路径

### Q: 导出的Excel格式不对？
**A**:
1. 使用Excel 2010+打开
2. 检查是否启用了宏
3. 确认Excel版本兼容性

## 标准查新问题

### Q: 查新结果不准确？
**A**:
1. 标准号格式要规范
2. 多个数据源交叉验证
3. 手动核对关键信息
4. 注意版本年份

### Q: 替代标准检测不到？
**A**:
1. 部分源可能不提供替代信息
2. 尝试多个数据源
3. 手动查询确认

### Q: 状态显示不对？
**A**:
1. 数据源信息可能有延迟
2. 以官方网站为准
3. 定期更新缓存

---

## 缓存问题

### Q: 如何清理缓存？
**A**:
```bash
方法1: 双击运行 scripts\启动应用（清理缓存）.bat
方法2: 手动删除 cache/ 目录
```

### Q: 缓存占用空间大？
**A**:
1. 定期清理缓存
2. 缓存会自动管理大小
3. 可以安全删除整个cache目录

### Q: 缓存数据过期？
**A**:
1. 清理缓存后重新搜索
2. 缓存默认有效期为7天
3. 可在配置中调整

---

## 打包问题

### Q: 如何打包成exe？
**A**:
```bash
双击运行: scripts\打包程序.bat
```
生成的exe在 `dist/MultiSourceDownloader/` 目录

### Q: 打包失败？
**A**:
1. 确认PyInstaller已安装
2. 检查磁盘空间（需要约500MB）
3. 关闭杀毒软件
4. 查看错误日志

### Q: exe文件很大？
**A**:
1. 正常情况约100-200MB
2. 包含了Python运行时和所有依赖
3. 可以压缩后分发

### Q: exe在其他电脑无法运行？
**A**:
1. 确认目标系统是Windows 7+
2. 安装VC++ Redistributable
3. 检查杀毒软件是否拦截
4. 以管理员权限运行

---

## 性能问题

### Q: 程序占用内存大？
**A**:
1. 正常占用约200-300MB
2. 批量处理时会增加
3. 关闭不用的功能
4. 定期重启程序

### Q: CPU占用高？
**A**:
1. 搜索和下载时会占用CPU
2. ZBY源使用浏览器，占用较高
3. 完成后会自动降低
4. 减少并发任务

---

## 其他问题

### Q: 如何更新程序？
**A**:
1. 从GitHub下载最新版本
2. 覆盖旧文件（保留config和cache）
3. 重新安装依赖

### Q: 数据安全吗？
**A**:
1. 所有数据存储在本地
2. 不上传任何信息到服务器
3. 缓存可以随时清理

### Q: 支持哪些标准格式？
**A**:
- 国家标准：GB、GB/T
- 行业标准：JB、HG等
- 其他标准：根据数据源支持情况

### Q: 可以离线使用吗？
**A**:
- 搜索和下载需要网络
- 已下载的文件可离线查看
- 缓存的结果可离线浏览

## 错误代码

| 错误代码 | 含义 | 解决方案 |
|---------|------|---------|
| 网络超时 | 连接超时 | 检查网络，增加超时时间 |
| 404 | 文件不存在 | 尝试其他数据源 |
| 403 | 访问被拒绝 | 检查配置，稍后重试 |
| 500 | 服务器错误 | 稍后重试，或换其他源 |
